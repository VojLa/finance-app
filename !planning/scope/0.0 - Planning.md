## Prijata zakladni rozhodnuti

- Python a FastAPI jako primarni backend,
- PostgreSQL jako hlavni databaze,
- modularni monolit jako vychozi architektura,
- oddeleni read modelu od canonical historie,
- generovani TypeScript kontraktu z OpenAPI,
- prevod vlastnictvi DB schema z Prisma na SQLAlchemy a Alembic.

## Otevrene otazky pro pozdejsi faze

Nasledujici rozhodnuti neblokuji zahajeni `0.1` a budou uzavrena az podle konkretni implementacni potreby:

- konkretni auth a session provider,
- konkretni job queue a broker,
- produkcni hosting a secrets manager,
- konecna observability platforma,
- pouziti Rustu pro konkretni vykonove hotspoty.
