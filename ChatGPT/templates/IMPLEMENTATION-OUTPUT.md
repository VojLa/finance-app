# Výstup implementace

## Výsledek

2–5 vět: co se uživatelsky nebo systémově změnilo.

## Změněné soubory

U každého souboru jednou větou uveď důvod změny. Neopisuj diff.

## Splněná kritéria

Zkopíruj akceptační kritéria ze STEP a označ splněno/nesplněno. Nesplněné body neskrývej.

## Ověření

| Kontrola | Výsledek |
|---|---|
| Relevantní test | PASS/FAIL + příkaz |
| Ruff | PASS/FAIL/NEBĚŽELO |
| Formát | PASS/FAIL/NEBĚŽELO |
| mypy | PASS/FAIL/NEBĚŽELO |
| pytest/quality gate | PASS/FAIL/NEBĚŽELO |
| DB/API kontrola | PASS/FAIL/NEAPLIKUJE SE |

## Bezpečnost a data

Potvrď autorizaci/account isolation, validaci vstupů, zacházení s citlivými daty, transakční/rollback chování a peněžní přesnost. Uveď skutečné testy, ne obecné tvrzení.

## Odchylky a rizika

Uveď změny proti plánu, známá omezení, dočasné adaptéry a navazující práci. Pokud nic nezůstává, napiš `Žádné známé`.
